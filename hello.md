git : 프로그램과 관련된 파일을 저장
clone
add: 스테이지에 올리기 (저장을 원하는 파일을 묶는다.)
commit: 저장 규칙: 수정 한번에 한번씩
push: github에 업로드 
코드뭉치 버리기: 마지막 저장으로 돌아가기
병합: 두 브렌치 일치시키기 
병합할때 충돌이 나면
    수동으로 제거(커밋 메세지 자동임)
    충돌 해결 버튼
reset: 이전 커밋으로 되돌리기(비추)
    (위의 커밋 전부 사라짐)
    (강제 푸시 해야됨)-> 비추
branch 만들어서 되돌리기 (추천!)
    내용이 사라지지 않음
revert 이용해서 되돌리기
    내용이 사라지지 않음
